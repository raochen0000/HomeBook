# Part 1 · 视觉设计规范

## 1. 设计原则与基调

### 1.1 气质关键词

**克制（Calm）· 清晰（Clear）· 原生（Native）**。核心原则：**视觉服从内容**——金融数据要「理性、精确、可信」，界面就该安静、不抢戏。参照 iOS「设置 / 提醒事项」：大量留白、分组列表为主、系统材质 chrome、装饰极少。

### 1.2 五条基调铁律（本版核心，AI 必须遵守）

1. **骨架黑白灰**：页面底、卡片、文本、分隔线**只用黑 / 白 / 灰阶**，不设主题色 / 品牌色 / 家庭温暖色。
2. **交互统一蓝**：一切「可点 / 选中 / 高亮 / 链接 / 主操作」用**系统蓝**（iOS systemBlue），Light / Night 自动切换。蓝是**唯一**的交互强调色。
3. **收支红绿**：金额语义色仅 **红（收入）/ 绿（支出）**，且**必须**同时配 `+/-` 符号与文案，不能只靠颜色区分。
4. **分类彩底**：分类图标用**低饱和彩色圆底**做功能识别（类比「提醒事项」每个清单的彩色图标圆），是识别色、非品牌色，纯前端映射不入库。
5. **分区靠明度**：采用 **iOS 设置式分组**——**浅灰页面底 + 纯白卡片**，靠明度差分区；**阴影只留给真正的浮层**（FAB / Toast / Sheet），普通卡片不投影。

> **为何选「灰底白卡」而非「白底 + 阴影白卡」**：阴影在深色模式几乎不可见，会逼出一套额外的「抬升表面色」规则；而灰底白卡在 Light / Night **两种模式都天然成立**，且与现有实现（`@expo/ui` List `insetGrouped`）一致，最省事、最原生。

### 1.3 与流程图配色的关系

PRD §2.4 的配色仅用于**流程图（Mermaid）绘制**，与界面视觉无关，两者互不混用。

> 落地优先级：**iOS 系统语义色（原生件默认）> 本表 hex 值**。系统语义色自动适配 Light / Night 与对比度；需精确控制时才用 hex。完整落地方式见附录 A.4。

### 2.1 基调：黑 / 白 / 灰

| 角色   | Light                     | Night                | 承载                           |
| ------ | ------------------------- | -------------------- | ------------------------------ |
| **黑** | `#000000`（文本 / 图标）  | 反相为白             | 主文本、主图标、金额           |
| **白** | `#FFFFFF`（卡片）         | 反相为近黑 `#1C1C1E` | 卡片、列表行、浮层内底         |

### 2.5 强调 / 交互色（蓝 · 系统 tint）

> **本版关键变更**：交互强调色由旧版「近黑 / 近白」改为**系统蓝**。凡「可点、选中、高亮、链接、主操作」皆用蓝。

| 令牌               | Light                  | Night                   | 用途                                                                 |
| ------------------ | ---------------------- | ----------------------- | -------------------------------------------------------------------- |
| `accent/primary`   | `#007AFF`              | `#0A84FF`               | 选中态、链接、可点文字、聚焦框、进度条常态、系统导航（**不再做按钮实底**，v0.6.5） |
| `accent/onPrimary` | `#FFFFFF`              | `#FFFFFF`               | 蓝底上的文字 / 图标                                                  |
| `accent/tint`      | `rgba(0,122,255,0.12)` | `rgba(10,132,255,0.18)` | 选中行浅底、选中 chip、浅蓝填充                                      |

**蓝色用到的地方（清单）**：选中态（列表选中行 / Menu 勾选 `checkmark`）、可点文字与链接（含列表「＋ 添加 / 新建」行）、开关 on（见下注）、线性 / 环形进度的常态色、输入聚焦框、返回箭头等系统导航元素。

> **v0.6.5 关键变更**：**按钮实底不再用蓝，改用墨色 `ink`**（详见下方例外 ③）。主 CTA、➕ 记一笔 FAB、Sheet 顶部确认文字动作（完成 / 保存）全部转墨；蓝退回「跳转 / 链接 + 系统交互」。

> **蓝色的例外（2026-07-01 校准）**：① **分段控件**用 iOS 原生默认（白色选中滑块，非蓝，见 §9.6）；② **分类圆底选中态**用**灰色描边**（`gray/500`，2px）而非蓝，避免与彩色圆底抢色（见 §2.8）。

> **蓝色的例外 ③ · 墨色 `ink`（2026-07-17 起，v0.6.5 扩为全局）**：一切「**用户的确认 / 主操作**」实底用 `ink`（随主题反相：浅色近黑 / 深色近白），而非蓝。范围：
> - **主 CTA 实底**（登录、重置密码、保存、提交反馈、加入 / 创建家庭、绑定 / 换绑、设置预算、新建目标、存入…）；
> - **➕ 记一笔 FAB 实底**；
> - **Sheet 顶部的确认文字动作**——完成 / 保存 / 应用 / 获取验证码——`text/primary` + 加粗（`取消` 用 `text/secondary`）；
> - 勾选态（协议、记住我）、滑动确认控件（非破坏性时）。
>
> **为何这么定（2026-07-19 与用户确认）**：登录页早已用墨色胶囊，而紧贴它弹出的「忘记密码」sheet 的主按钮却是蓝——同一视觉动线里黑蓝混用，很刺眼。把「确认动作」整体收敛到墨色后，一条规则贯穿全 app：**墨 = 你的确认动作，蓝 = 跳转 / 链接，红 = 破坏**。蓝不再兼任按钮实底，语义只剩「可点 / 跳转 / 选中」，层级更干净。
>
> **边界（必须守住）**：蓝仍是链接与系统交互色——可点文字、《用户协议》一类链接、列表「＋ 添加 / 新建」行、列表选中态与 `checkmark`、开关 on、进度、聚焦框——这些**不要**改墨。破坏性动作走红（`state/danger`），不走墨。
>
> ⚠️ **与本节开头历史决议的关系**：本节开头曾把交互色由更早的「近黑 / 近白」整体改为系统蓝；v0.6.5 只把其中「**按钮 / 确认动作**」这一类拨回墨色，**选中 / 链接 / 系统交互仍是蓝**，并非整体回退到近黑近白管一切。

> **开关（Toggle）颜色**：本规范定 **on = `accent/primary`（蓝）**，与「交互统一蓝」一致；若需更贴近 iOS 默认，可改 systemGreen 绿（二选一，全局统一即可）。

## 3. 明暗模式（Light / Night）

- 统一提供 **Light / Night** 两种模式，跟随系统或手动切换（我的页 → 深色模式）。
- 深色模式切换须**真正驱动全局** `palette` / `colorScheme`，不是单页视觉。
- **Night 分区不靠阴影**（深底上阴影不可见），靠 `bg/base`（纯黑）↔ `bg/card`（近黑 `#1C1C1E`）的明度差 + 分隔线；浮层用更亮一档表面色（`bg/elevated` `#2C2C2E`）。
- 蓝 / 红 / 绿 / 彩底在 Night 全部使用降明度值（见各表 Night 列）。

### 3.0 深色模式四条铁律（2026-07-17 新增 · AI 必须遵守）

> **立规起因**：登录页主按钮与勾选态在深色下整个消失。根因不是「深色没定颜色」，而是组件里写死了 `#1C1C1E`，而它**逐字等于** `bg/card` 的 Night 值——按钮底和它所在的卡片同色，对比度 **1.00:1**。浅色下它是白卡上的黑胶囊、好看得很，所以一直没人发现。下面四条各自封死这个 bug 的一个环节。

**铁律 1 · 组件内禁止裸 hex，颜色一律取自 `usePalette()`。**
写死的 hex 只在你当时看的那个模式里成立，另一个模式是碰运气。

_豁免（唯一判据：**该颜色所在的表面本身就是主题无关的固定色**，此时它没有「另一个模式」可言）_：
- **彩底上的白**——分类圆底 / 成员头像内的图标与文字恒为 `#FFFFFF`（§2.8 已明确规定）。圆底色来自 `cat/*` 或头像色，不随主题翻转，故其上的白也不需要翻转。
- **二维码前后景**（`invite-sheet.tsx`）——必须强制纯黑白才能被扫。
- **`shadowColor`**——阴影恒为黑，且深色下本就不可见（见本节第三条）。

不属于以上三类的 `#RRGGBB`，出现在 `src/features/` 或 `src/app/` 里即为待修。

**铁律 2 · 填充色不得等于它所在的表面色，且须与之差 ≥ 3:1。**
"按钮底 = 卡片底" 这种写法必须在 review 阶段就被打回。自检问法：**这个填充坐在哪一层表面（base / card / elevated）上？它和那一层的对比度是多少？**

**铁律 3 · 非文字控件也要 ≥ 3:1（WCAG 1.4.11），不只是文字。**
§11 原先只约束了文本对比度，所以一个 1.00:1 的按钮完全合规地溜了过去。按钮底、勾选态、图标、聚焦框、进度条，全部纳入。

**铁律 4 · 深色不是把浅色反相，两个模式各自成立、各自验证。**
每个语义令牌必须有独立的 Light / Night 取值。**同一个 hex 在两个模式共用是 code smell**——要么它真的是主题无关（罕见，如二维码），要么它就是下一个 bug。需要"随主题翻转"的语义（如 `ink`），把翻转做进令牌里，而不是让组件去 `isDark ? a : b`。

> **顺带（已在做，写下来免得丢）**：深色下大面积纯白会光晕（halation），故 `ink` 的 Night 取 `#F5F5F7` 而非 `#FFFFFF`；饱和色在深色下一律提亮 + 降饱和（`#E2563D` → `#FF7461`）。

## 4. 字体与排版

- **字体**：中文 PingFang SC，英文 / 数字 SF Pro；**全程 Dynamic Type**。
- **金额必须等宽数字**（Tabular / monospaced digit）——记账 App 硬规则。
- **偶数化规则**：自定义字号 / 行高若为奇数，统一向下取偶数；使用系统 text style 时仍优先保留 Dynamic Type 缩放能力。

| 令牌               | 用途                     | 字号 | 行高 | 字重               |
| ------------------ | ------------------------ | ---- | ---- | ------------------ |
| `type/largeTitle`  | 大金额（结余、目标进度） | 34   | 40   | Bold（数字）       |

## 5. 金额展示规范（App 主角）

- **符号**：支出 `-`、收入 `+`，取语义色；结余 / 中性取 `text/primary`。
- **整数 / 小数分级**：整数部分主字号主字重；**小数部分（含小数点）降一档字号 + 字重**（hero 40→28、Bold→Regular）。
- **千分位**：每三位加分隔；**等宽数字必开**。
- **两位小数恒定**：金额始终展示两位小数（不提供小数开关）。
- **币种位预留**：货币符号在金额左侧独立槽位（多币种远期）。

## 6. 间距、网格与圆角

- **基础栅格：8pt 网格**；本版 `space/*` 在旧刻度上整体 -2pt，形成更紧凑的 2pt 微调刻度。

| 令牌       | 数值 | 典型用途                |
| ---------- | ---- | ----------------------- |
| `space/1`  | 2    | 图标-文字间隙           |
| `space/2`  | 6    | 元素内间距              |
| `space/3`  | 10   | 行内分组                |
| `space/4`  | 14   | **页边距 / 卡片内边距** |
| `space/5`  | 18   | 卡片间距                |
| `space/6`  | 22   | 分区间距                |
| `space/8`  | 30   | 大分区                  |
| `space/10` | 38   | 空状态留白              |
| `space/12` | 46   | 顶部 hero 留白          |

| 圆角令牌      | 数值 | 用途                            |
| ------------- | ---- | ------------------------------- |
| `radius/sm`   | 8    | 标签、小按钮                    |
| `radius/md`   | 12   | 输入框、小卡片、主按钮 fallback |
| `radius/lg`   | 16   | 内容卡片（默认）                |
| `radius/lg+`  | 20   | 大内容卡片 / 目标卡             |
| `radius/xl`   | 28   | 模态面板（Sheet）顶部           |
| `radius/full` | 9999 | 头像、进度环、圆形元素          |

- **列表行高 ≥ 44pt；最小可点击区域 44×44pt**。
- **圆角规范核对**：Apple HIG 建议按钮优先使用系统定义的尺寸相关圆角；新设计系统强调 fixed / capsule / concentric 三类形状。当前 `8 / 12 / 16 / 20 / 28 / full` 作为自绘 fallback 梯度合理，分别覆盖标签、输入、小卡片、内容卡片、大卡片、Sheet 与圆形元素。
- **按钮与系统件优先级**：原生 Button / List / Sheet 优先使用系统默认圆角；主操作按钮若为大触控控件，可使用系统按钮默认半径或胶囊形，不强行覆盖为固定数值。
- **同心圆角（iOS 26）**：嵌套容器遵循同心关系；用原生容器时由系统计算，必须自绘时以内圆角 ≈ 外圆角 − 内边距为准，避免内外圆角过挤或外扩。

### 9.9 浮层分层：Sheet / 弹窗 / push 页 / 二次确认

**先判层级——按「用户要不要回头」从上往下，第一个命中就用：**

| 层                   | 形态                                 | 用于                                                             | 例                                             |
| -------------------- | ------------------------------------ | --------------------------------------------------------------- | ---------------------------------------------- |
| ① 系统弹窗 / 操作单  | `Alert` / `ActionSheet`              | **无输入**，只告知或选是否 / 选一个操作                         | 删除确认、转让 / 移除成员菜单                   |
| ② 系统 pageSheet     | 半屏升起、接近满屏、下滑关           | 有始有终的小任务，做完回原页、上文要留着；**内容高 / 可滚动 / 表单类** | 记账、预算、成员管理、通知中心、协议隐私、忘记密码 |
| ③ push 独立页        | 进栈整页、自带返回头（**非 Modal 全屏**） | 用户会**久留、来回翻、或内部还有层级**                          | 月度总结（`/summary`）、搜索页                  |
| ④ 自绘·贴合内容矮卡  | 底部升起、高度贴合内容、抓手可下拉关 | **短的固定内容**，或**需叠在另一个 sheet 之上**                 | 流水详情、搜索筛选下拉、Apple 登录卡            |

**硬约束（AI 必须遵守）：**

- **pageSheet 不叠加**：绝不在一个 pageSheet 里再开另一个 pageSheet（iOS 会二次内缩、两层都能下滑关，用户不知在关哪层）。需要子层级时——**单壳内 view 切换**（分类管理：列表 ↔ 编辑器）或**先关本页、由父层在其 `onDismiss` 后打开下一个**（成员管理 →邀请）。
- **内容高度决定 ② 还是 ④**：RN `Modal` 没有 detent / 自适应高度 API，系统 pageSheet 只能接近满屏。短内容硬套 pageSheet 会在底部留大片空白 → 该用 ④ 自绘矮卡；高 / 可滚动内容才用系统 pageSheet。搜索筛选下拉、Apple 登录卡、流水详情自绘皆因此（同一物理约束，非随意选择）。
- **④ 的做法**：顶部 `radius/xl` + **抓手**（抓手要真能下拉关闭，不是装饰——`translateY` 跟手、下拉超阈值即关）+ 系统遮罩（`opacity/scrim`）+ 高度贴合内容、底部只留安全区（`insets.bottom`）。下拉关闭收尾**不要把卡片拉回原位再交给系统动画**，否则会出现「二次展示」的闪影（让卡片停在屏幕外、由 Modal 滑出无痕收尾；复位放到下次打开前用 `useLayoutEffect` 同步做）。

**外观细节：**

- **Sheet 圆角**：优先系统 Sheet 圆角；自绘 fallback 顶部 `radius/xl` + 抓手 + 系统遮罩。记账面板等大号内容用 `.large` detent（§10.3）。
- **内容型 pageSheet**（纯阅读内容，如用户协议 / 隐私政策）：除抓手外，**右上角加 X 关闭按钮**（`xmark`，圆形浅底 `fill/tertiary`），提供显式关闭入口；操作型 Sheet（记账 / 表单）以抓手 + 动作按钮收尾即可。登录页与「关于家账」的协议 / 隐私共用此内容型 pageSheet（PRD §3.6 / §18.3.8）。（注：④ 的短内容矮卡靠抓手下拉关，不另加 X。）
- **二次确认**：破坏性项用系统确认弹窗，破坏性动作红色（`role: destructive`）。

### 9.10 反馈 / 空状态 / 骨架

| 元素            | 外观                                                                                   |
| --------------- | -------------------------------------------------------------------------------------- |
| 顶部条幅 Banner | 中性 `bg/card` 底；仅用于非预警轻通知（如「上月总结来啦」）。预警内联进脉搏卡（§10.2） |

## 11. 可访问性（强约束）

- 文本对比度 ≥ WCAG AA（正文 4.5:1，大字 3:1）；Light / Night 各自验证。
- **非文字控件对比度 ≥ 3:1**（WCAG 1.4.11）：按钮实底、勾选 / 选中态、图标、聚焦框、进度条，均须与**相邻表面色**达标；Light / Night 各自验证。参见 §3.0 铁律 2 / 3——只查文本对比度会漏掉「按钮底 = 卡片底」这类 1.00:1 的缺陷。
- **收支不只靠颜色**：必带 `+/-` 与文案；分类色为辅助识别，必带名称。
- **Dynamic Type、VoiceOver、减弱动态、Light / Night** 全面支持。
- 触控目标 ≥ 44×44pt。

## 附录 A · `@expo/ui/swift-ui` 工程实现映射

> 本附录是「怎么用原生件实现正文的视觉」，与视觉外观无关。喂 AI 生成原型时**无需**本附录。

### A.1 实现策略：Native-First

- **平台基线**：iOS App（iOS 26+），**React Native（Expo SDK 56）+ TypeScript**，UI 主体由 `@expo/ui/swift-ui` 渲染**真实原生 SwiftUI 视图**，视觉对齐 iOS 26 HIG。
- **优先级（高→低）**：
  1. 能用 `@expo/ui/swift-ui` 原生件的一律用原生件（Button / List / Form / Section / TabView / BottomSheet / Picker / DatePicker / Toggle / Slider / ProgressView / Gauge / Text / Image 等）。
  2. 需调样式的经 `modifiers`（`@expo/ui/swift-ui/modifiers`）调整，不绕过框架。
  3. 库确实做不到的（自定义数字键盘、报表图表、滑动确认）才退回自定义 RN / react-native-svg / Reanimated，经 `RNHostView` / `Host` 边界拼接。
- **判定顺序**：先问「`@expo/ui` 有没有对应组件？」→ 有则用 → 样式不够加 modifier →（仅在）确实不行 → 自定义并在 A.5 登记。

### A.2 三类实现层

| 层                    | 渲染方式                             | 令牌如何作用                      | 典型界面                             |
| --------------------- | ------------------------------------ | --------------------------------- | ------------------------------------ |
| **L-Native 原生层**   | `@expo/ui/swift-ui` 原生 SwiftUI 件  | 经 `modifiers` 传入（见 A.4）     | 设置 / 家庭 / 列表、按钮、开关、进度 |
| **L-Bridge 桥接层**   | 原生件包裹自定义内容（`RNHostView`） | 外壳走原生 modifier，内部走自定义 | 记账 Sheet（原生壳 + 自定义键盘）    |
| **L-Custom 自定义层** | RN `View`/`Text` / react-native-svg / Reanimated | 令牌全量生效（直接读 theme）      | 报表图表、滑动确认、金额大数键盘     |

> **工程纪律**：进入 `Host` 内部后**无 flexbox / Yoga**，布局用 `HStack` / `VStack` / `Spacer` + `spacing`；自定义层才回到 RN flexbox。

### A.3 材质与 Liquid Glass

| 区域                                             | 材质规则                                            |
| ------------------------------------------------ | --------------------------------------------------- |
| **系统 chrome**（导航 / Tab Bar / Sheet 外壳）   | **顺应 iOS 26 系统材质**（含 Liquid Glass），不覆盖 |
| **内容层**（卡片、流水行、报表、金额、表单分组） | **保持实心**：不主动施加 `glassEffect`              |

> 一句话：系统外壳交给系统（含玻璃），内容主体保持实心，不额外使用 `glassEffect`。

### A.4 令牌可达性矩阵

| 设计令牌类别                | 落地方式（modifier / prop）                                                          | 可达性       |
| --------------------------- | ------------------------------------------------------------------------------------ | ------------ | ------------------- | --------- | ------- |
| 背景 `bg/*`、`accent/*` 底  | `background('#hex')`（或让 `Form`/`List` 用系统默认）                                | ✅ 完全      |
| 文本 `text/*`、`semantic/*` | `foregroundColor('#hex')`；次要文本优先 `foregroundStyle({hierarchical, secondary})` | ✅ 完全      |
| 圆角 `radius/*`             | `cornerRadius(n)` / `clipShape('roundedRectangle'                                    | 'circle')`   | ✅ 完全             |
| 间距 `space/*`              | `padding({...})` + `HStack/VStack spacing`                                           | ✅ 完全      |
| 海拔 `elevation/*`          | `shadow({ radius, x, y, color })`（Night 降 opacity）                                | ✅ 完全      |
| 尺寸                        | `frame({ width, height })`                                                           | ✅ 完全      |
| 字阶 `type/*`               | `font({ size, weight, design })`；等宽 `design:'monospaced'`                         | ✅ 大部分    |
| 按钮样式                    | `buttonStyle('plain'                                                                 | 'bordered'   | 'borderedProminent' | 'glass')` | ✅ 完全 |
| 分类彩底                    | `frame` + `background('#hex')` + `clipShape`                                         | ✅ 完全      |
| 描边                        | `overlay` + 自定义边                                                                 | ⚠️ 部分      |
| 动效 `motion/*`             | `animation(Animation.spring/..., dep)`；复杂走自定义                                 | ✅ 部分      |
| 玻璃 `glassEffect`          | 内容层不使用；仅系统 chrome 由系统负责                                               | ✅（限场景） |

> **走自定义层的**：报表图表、自定义数字键盘、滑动确认、复杂自绘进度。

### A.5 组件落地映射表（界面元素 → 原生件 → 何时自定义）

### A.6 关键 modifier 速记

```
background('#hex')                                            背景色
foregroundColor('#hex') / foregroundStyle({hierarchical,secondary})  文本/图标色
cornerRadius(n) / clipShape('roundedRectangle'|'circle')     圆角/裁形
padding({ all|horizontal|vertical|top|... })                 内边距
frame({ width, height })                                     固定尺寸
shadow({ radius, x, y, color })                              阴影（仅浮层）
font({ size, weight, design:'monospaced' })                  字阶 / 等宽数字
buttonStyle('plain'|'bordered'|'borderedProminent'|'glass')  按钮样式

animation(Animation.spring({duration}), dep)                 原生动效
```

> 布局：进入 `Host` 后无 flexbox，用 `HStack`/`VStack`/`Spacer` + `spacing`；跨回 RN 自定义内容经 `RNHostView` 或重新包 `Host`。
