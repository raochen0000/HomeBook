# Part 1 · 视觉设计规范

## 1. 设计原则与基调

### 1.1 气质关键词
**克制（Calm）· 清晰（Clear）· 原生（Native）**。核心原则：**视觉服从内容**——金融数据要「理性、精确、可信」，界面就该安静、不抢戏。参照 iOS「设置 / 提醒事项」：大量留白、分组列表为主、系统材质 chrome、装饰极少。

### 1.2 五条基调铁律（本版核心，AI 必须遵守）

1. **骨架黑白灰**：页面底、卡片、文本、分隔线**只用黑 / 白 / 灰阶**，不设主题色 / 品牌色 / 家庭温暖色。
2. **交互统一蓝**：一切「可点 / 选中 / 高亮 / 链接 / 主操作」用**系统蓝**（iOS systemBlue），Light / Night 自动切换。蓝是**唯一**的交互强调色。
3. **收支红绿**：金额语义色仅 **红（收入）/ 绿（支出）**，且**必须**同时配 `+/-` 符号与文案，不能只靠颜色区分。
4. **分类彩底**：分类图标用**低饱和彩色圆底**做功能识别（类比「提醒事项」每个清单的彩色图标圆），是识别色、非品牌色，纯前端映射不入库。
5. **分区靠明度**：采用 **iOS 设置式分组**——**浅灰页面底 + 纯白卡片**，靠明度差分区；**阴影只留给真正的浮层**（FAB / Toast / Sheet），普通卡片不投影。

> **为何选「灰底白卡」而非「白底 + 阴影白卡」**：阴影在深色模式几乎不可见，会逼出一套额外的「抬升表面色」规则；而灰底白卡在 Light / Night **两种模式都天然成立**，且与现有实现（`@expo/ui` List `insetGrouped`）一致，最省事、最原生。

### 1.3 与流程图配色的关系
PRD §2.4 的配色仅用于**流程图（Mermaid）绘制**，与界面视觉无关，两者互不混用。

## 3. 明暗模式（Light / Night）

- 统一提供 **Light / Night** 两种模式，跟随系统或手动切换（我的页 → 深色模式）。
- 深色模式切换须**真正驱动全局** `palette` / `colorScheme`，不是单页视觉。
- **Night 分区不靠阴影**（深底上阴影不可见），靠 `bg/base`（纯黑）↔ `bg/card`（近黑 `#1C1C1E`）的明度差 + 分隔线；浮层用更亮一档表面色（`bg/elevated` `#2C2C2E`）。
- 蓝 / 红 / 绿 / 彩底在 Night 全部使用降明度值（见各表 Night 列）。

## 5. 金额展示规范（App 主角）

- **符号**：支出 `-`、收入 `+`，取语义色；结余 / 中性取 `text/primary`。
- **整数 / 小数分级**：整数部分主字号主字重；**小数部分（含小数点）降一档字号 + 字重**（hero 40→28、Bold→Regular）。
- **千分位**：每三位加分隔；**等宽数字必开**。
- **两位小数恒定**：金额始终展示两位小数（不提供小数开关）。
- **币种位预留**：货币符号在金额左侧独立槽位（多币种远期）。

## 7. 海拔与阴影（克制 · 仅浮层）

> 采用灰底白卡分区后，**普通卡片不投影**。阴影**只留给真正脱离页面的浮层**。

| 令牌          | 阴影规格                         | 用途                                     |
| ------------- | -------------------------------- | ---------------------------------------- |
| `elevation/0` | 无阴影，靠灰白明度差 + `Divider` | **所有分组卡 / 列表行 / 内容卡（默认）** |
| `elevation/1` | `shadow(radius:8, y:1, 黑 8%)`   | ➕ FAB、Toast、悬浮顶栏                   |
| `elevation/2` | `shadow(radius:16, y:2, 黑 10%)` | 强浮起元素（少用）                       |

> **系统 chrome / Sheet / 弹窗**的海拔由系统材质负责，不手动加阴影。**Night** 阴影 opacity 降到 ≤ 4%，改用更亮表面色（`bg/elevated`）表达海拔。

## 8. 图标系统

- **系统 / 功能图标**：一律 **SF Symbols**（自动适配粗细 / 深浅色）。
- **分类图标**：统一描边、单色可着色图标族 + §2.8 低饱和彩色圆底。
- **插画**：**不使用暖色手绘插画**；空状态等改用 SF Symbols 大图标（单色 `text/tertiary`）。

| 令牌        | 数值 | 用途           |
| ----------- | ---- | -------------- |
| `icon/sm`   | 16   | 行内角标、辅助 |
| `icon/md`   | 20   | 列表次要图标   |
| `icon/base` | 24   | Tab、通用图标  |
| `icon/lg`   | 28   | 分类圆底内图标 |

### 8.1 头像规则

- 无自定义头像时用**系统默认头像占位**，**不取用户名首字生成字符头像**。
- 圆形（`radius/full`），用于顶栏入口、记账人标记、成员列表、成员贡献图、加入家庭预览卡。
- 多人家庭显示记账人头像。

## 9. 组件视觉规格（通用词汇）

> 本节定义**组件外观词汇**；§10 的页面规格直接引用这些词汇。每个组件的原生件选型见附录 A.5。

### 9.1 顶栏 / 导航栏
- 左上 = 当前 Tab 名**大标题**（「我的」页除外，顶部为个人资料头）；右上 = 🔍 搜索图标（`magnifyingglass`）。
- 顺应 iOS 26 系统材质；滚动时标题收起、顶栏合并为悬浮细条（`elevation/1`）。

### 9.2 Tab Bar + 记一笔 FAB
- **四 Tab**：首页(`house`) / 报表(`chart.pie`) / 家庭(`person.2`) / 我的(`person.crop.circle`)。顺应系统材质。
- **➕ FAB**：圆形 56×56，`accent/primary`（**蓝**）实底 + `accent/onPrimary` ➕（`plus`），`elevation/1` 轻投影；固定 Tab Bar 右上方、全 Tab 常驻；右边距 16。点击 → 弹记账面板（§10.3）。

### 9.3 分组卡 / Section
- `bg/card` 白卡 + `radius/lg`，浮于 `bg/base` 灰底；卡内多行用 `separator` 分隔、左缩进对齐标签。
- 分组头（可选）：`type/footnote`、`text/secondary`、卡上方左侧。

### 9.4 列表行

| 类型         | 外观                                                                  |
| ------------ | --------------------------------------------------------------------- |
| **默认行**   | 图标底片 + 标签 + 弹性空隙 + 行尾灰值（可选）+ `chevron.right`；点击 push |
| **下拉型行** | 行尾为下拉菜单，label = 当前值 + `chevron.up.chevron.down`；**无** `chevron.right` |

| **危险行**   | 文字取 `state/danger`（红），如「退出登录 / 账号注销」                |
| **选中行**   | 行尾 `checkmark`（蓝）或整行浅蓝底 `accent/tint`                      |

### 9.5 按钮

| 类型     | 外观                                                                    |
| -------- | ----------------------------------------------------------------------- |
| 主 CTA   | `accent/primary`（蓝）实底 + `accent/onPrimary` 文字 + `radius/md`；禁用降透明度 |
| 次要按钮 | `accent/tint` 浅蓝底 / 描边 + `accent/primary` 文字                      |

| 破坏性   | `state/danger`（红）文字 / 底                                            |

### 9.7 表单控件

| 日期     | 紧凑日期选择器（日期 + 时分）                                    |
| 输入框   | `bg/card` 底 + `radius/md`，聚焦框 `accent/primary`（蓝）        |

| 元素     | 外观                                                                            |

### 9.9 Sheet / 弹窗 / 二次确认
- **Sheet**：从底部升起，顶部 `radius/xl` 圆角 + 抓手 + 系统遮罩（`opacity/scrim`）；大号内容用 `.large` detent。
- **二次确认**：破坏性项用系统确认弹窗，破坏性动作红色（`role: destructive`）。

### 9.10 反馈 / 空状态 / 骨架

| 元素            | 外观                                                                      |
| --------------- | ------------------------------------------------------------------------- |
| 顶部条幅 Banner | 中性 `bg/card` 底；仅用于非预警轻通知（如「上月总结来啦」）。预警内联进脉搏卡（§10.2） |

### 9.11 滑动操作（列表行左滑）

## 附录 A · `@expo/ui/swift-ui` 工程实现映射

> 本附录是「怎么用原生件实现正文的视觉」，与视觉外观无关。喂 AI 生成原型时**无需**本附录。

### A.1 实现策略：Native-First
- **平台基线**：iOS App（iOS 26+），**React Native（Expo SDK 56）+ TypeScript**，UI 主体由 `@expo/ui/swift-ui` 渲染**真实原生 SwiftUI 视图**，视觉对齐 iOS 26 HIG。
- **优先级（高→低）**：
  1. 能用 `@expo/ui/swift-ui` 原生件的一律用原生件（Button / List / Form / Section / TabView / BottomSheet / Picker / DatePicker / Toggle / Slider / ProgressView / Gauge / Text / Image 等）。
  2. 需调样式的经 `modifiers`（`@expo/ui/swift-ui/modifiers`）调整，不绕过框架。
  3. 库确实做不到的（自定义数字键盘、报表图表、滑动确认）才退回自定义 RN / Skia / Reanimated，经 `RNHostView` / `Host` 边界拼接。
- **判定顺序**：先问「`@expo/ui` 有没有对应组件？」→ 有则用 → 样式不够加 modifier →（仅在）确实不行 → 自定义并在 A.5 登记。

### A.2 三类实现层

| 层                    | 渲染方式                             | 令牌如何作用                        | 典型界面                               |
| --------------------- | ------------------------------------ | ----------------------------------- | -------------------------------------- |
| **L-Native 原生层**   | `@expo/ui/swift-ui` 原生 SwiftUI 件  | 经 `modifiers` 传入（见 A.4）       | 设置 / 家庭 / 列表、按钮、开关、进度   |
| **L-Bridge 桥接层**   | 原生件包裹自定义内容（`RNHostView`） | 外壳走原生 modifier，内部走自定义   | 记账 Sheet（原生壳 + 自定义键盘）      |
| **L-Custom 自定义层** | RN `View`/`Text` / Skia / Reanimated | 令牌全量生效（直接读 theme）        | 报表图表、滑动确认、金额大数键盘       |

> **工程纪律**：进入 `Host` 内部后**无 flexbox / Yoga**，布局用 `HStack` / `VStack` / `Spacer` + `spacing`；自定义层才回到 RN flexbox。

### A.3 材质与 Liquid Glass

| 区域                                              | 材质规则                                             |
| ------------------------------------------------- | ---------------------------------------------------- |
| **系统 chrome**（导航 / Tab Bar / Sheet 外壳）    | **顺应 iOS 26 系统材质**（含 Liquid Glass），不覆盖  |
| **内容层**（卡片、流水行、报表、金额、表单分组）  | **保持实心**：不主动施加 `glassEffect`               |

> 一句话：系统外壳交给系统（含玻璃），内容主体保持实心，不额外使用 `glassEffect`。

### A.4 令牌可达性矩阵

| 设计令牌类别                  | 落地方式（modifier / prop）                                             | 可达性       |
| ----------------------------- | ----------------------------------------------------------------------- | ------------ |
| 背景 `bg/*`、`accent/*` 底    | `background('#hex')`（或让 `Form`/`List` 用系统默认）                    | ✅ 完全      |
| 文本 `text/*`、`semantic/*`   | `foregroundColor('#hex')`；次要文本优先 `foregroundStyle({hierarchical, secondary})` | ✅ 完全 |
| 圆角 `radius/*`               | `cornerRadius(n)` / `clipShape('roundedRectangle'|'circle')`            | ✅ 完全      |
| 间距 `space/*`                | `padding({...})` + `HStack/VStack spacing`                              | ✅ 完全      |
| 海拔 `elevation/*`            | `shadow({ radius, x, y, color })`（Night 降 opacity）                   | ✅ 完全      |
| 尺寸                          | `frame({ width, height })`                                              | ✅ 完全      |
| 字阶 `type/*`                 | `font({ size, weight, design })`；等宽 `design:'monospaced'`            | ✅ 大部分    |
| 按钮样式                      | `buttonStyle('plain'|'bordered'|'borderedProminent'|'glass')`          | ✅ 完全      |
| 分类彩底                      | `frame` + `background('#hex')` + `clipShape`                            | ✅ 完全      |
| 描边                          | `overlay` + 自定义边                                                    | ⚠️ 部分      |
| 动效 `motion/*`               | `animation(Animation.spring/..., dep)`；复杂走自定义                    | ✅ 部分      |
| 玻璃 `glassEffect`            | 内容层不使用；仅系统 chrome 由系统负责                                  | ✅（限场景） |

> **走自定义层的**：报表图表、自定义数字键盘、滑动确认、复杂自绘进度。

### A.5 组件落地映射表（界面元素 → 原生件 → 何时自定义）

| 界面元素（§9/§10）        | 首选原生件                                    | 何时自定义                              |
| ------------------------- | --------------------------------------------- | --------------------------------------- |
| 页面导航                  | Expo Router Stack（`NavigationStack`）         | —                                       |
| 四 Tab                    | `NativeTabs`（4 × `Trigger`）                  | alpha 不稳时退自定义 RN Tab Bar         |
| 顶栏搜索 / 头像入口       | toolbar item + `Image`                         | —                                       |
| 分组卡 / 列表             | `Form` / `Section` / `List`（`insetGrouped`）  | 超大列表换 `@shopify/flash-list`        |
| ➕ FAB                    | 自定义悬浮层                                    | 本就是自定义                            |
| 按钮                      | `Button` + `buttonStyle` + `background`         | —                                       |
| 分段控件                  | `Picker` + `pickerStyle('segmented')`          | —                                       |
| 开关 / 下拉 / 日期        | `Toggle` / `Picker`·`Menu` / `DatePicker`      | —                                       |
| 二次确认 / 密码           | `ConfirmationDialog`（destructive）/ `SecureField` | —                                    |
| 分隔线                    | `Divider`                                       | —                                       |
| 线性 / 环形进度           | `LinearProgress`·`ProgressView` / `Gauge`      | 需精确变色 + 粗描边时自绘（Skia）       |
| 记账面板外壳              | 原生 `BottomSheet`（`.large` detent + 抓手）    | —                                       |
| 大号金额 + 数字键盘       | —                                              | **自定义层**（RN / Skia）经 `RNHostView` |
| 报表图表                  | —                                              | **自定义层**（`react-native-skia` 自绘） |
| 滑动确认 / 滑动操作       | `SwipeActions`（须在 `List` 内）              | 独立滑动确认控件走 Reanimated 手势      |
| 6 段邀请码 / OTP 输入     | —                                              | **自定义层**（无对应原生件）            |

> **背景反转注意**：正文 §2.3 的「灰底白卡」对应 iOS grouped 默认（`Form`/`List(.insetGrouped)` 自带灰底白卡），**直接用原生分组即可**，无需自绘卡片。

### A.6 关键 modifier 速记

```
background('#hex')                                            背景色
foregroundColor('#hex') / foregroundStyle({hierarchical,secondary})  文本/图标色
cornerRadius(n) / clipShape('roundedRectangle'|'circle')     圆角/裁形
padding({ all|horizontal|vertical|top|... })                 内边距
frame({ width, height })                                     固定尺寸
shadow({ radius, x, y, color })                              阴影（仅浮层）
font({ size, weight, design:'monospaced' })                  字阶 / 等宽数字
buttonStyle('plain'|'bordered'|'borderedProminent'|'glass')  按钮样式
opacity(n)                                                   不透明度（禁做玻璃）
animation(Animation.spring({duration}), dep)                 原生动效
```

> 布局：进入 `Host` 后无 flexbox，用 `HStack`/`VStack`/`Spacer` + `spacing`；跨回 RN 自定义内容经 `RNHostView` 或重新包 `Host`。

### A.7 Native-First 的红利与义务

| 免费红利（用原生件即得）                       | 自定义层的额外义务                         |
| ---------------------------------------------- | ------------------------------------------ |
| Dynamic Type、VoiceOver、Light/Night、同心圆角 | 须自行补齐无障碍标签、字号适配、深浅色切换 |
| iOS 26 一致材质与系统转场                      | 须自管材质一致性，勿与系统 chrome 脱节     |
| 系统语义色自动对比度达标                       | 须自测对比度（Light / Night 各一遍）       |
| `Form`/`List` 自带分组背景与分隔               | 自绘卡片须自管分隔与海拔                   |
