## 1. 设计语言与基线

- **平台基线**：iOS App，基于 **React Native（Expo）+ TypeScript** 构建，视觉对齐 **iOS 26 设计规范（HIG）**（技术选型详见 `TECH.md`）。
- **设计语言**：**不使用任何玻璃 / 半透明 / 模糊材质**。导航栏、Tab Bar、悬浮 ➕ 按钮、模态面板等「控件层」，与流水、报表、卡片等「内容层」一律采用**实心材质**，靠颜色层次、分隔线与极轻柔阴影区分海拔，保证金额数字清晰可读、跨设备一致、渲染性能稳定。
- **UI 模式**：统一提供 **Light（浅色）/ Night（深色）两种模式**，跟随系统或手动切换；所有设计令牌均提供两套映射（见 §4、§17）。
- **品牌气质关键词**：温暖（Warm）、克制（Calm）、归属感（Belonging）。
- **核心张力与解法**：金融数据要「理性、精确、可信」，家庭场景要「柔软、有温度」。解法 = **结构理性（8pt 网格、等宽数字、清晰层级）+ 表皮柔软（中性留白、圆角、暖色手绘插画、可选暖中性画布）**。
- **本规范与流程图配色的关系**：PRD §2.4 的配色仅用于**流程图（Mermaid）绘制**，与界面视觉无关；本规范定义的是**产品界面视觉**，两者相互独立，不得混用。

### 1.1 设计系统结构（如何读这份文档）

本规范采用业界通行的**三层令牌（tiered tokens）**结构，工程实现时按层落地，互不越级引用：

| 层  | 名称                   | 作用                                             | 示例                                          | 位置         |
| --- | ---------------------- | ------------------------------------------------ | --------------------------------------------- | ------------ |
| L1  | **原始令牌 Primitive** | 与语义无关的原始数值（调色板、刻度），与模式无关 | `gray/100`、`space/4`、`radius/lg`            | §4.1、§6、§8 |
| L2  | **语义令牌 Semantic**  | 表达「意图」，按 Light/Night 各映射一组 L1       | `text/primary`、`bg/base`、`semantic/expense` | §4.2、§5     |
| L3  | **组件令牌 Component** | 组件自身的属性，只引用 L2                        | `button.primary.bg`、`sheet.radius`           | §14          |

> **使用纪律**：UI 代码只消费 L3 / L2，**禁止在组件里直接写 L1 原始 hex**。引入品牌色或换肤时，只改 L2 映射，全局生效（见 §16）。

## 2. 色彩策略：界面坚持中性黑白（不引入品牌色）

### 2.1 决策

界面**不引入品牌色**，采用：

> **中性黑白骨架（含 Light / Night 两种模式）+ 收支语义色（珊瑚红 / 沉静绿）作为唯一的功能性色彩 + 分类图标低饱和彩色作为「呼吸口」。**

### 2.2 可行性依据（不会显得廉价）

1. **是成熟设计流派**：Apple 系统自带 App、Things、Notion 等大量采用「中性骨架 + 极少强调色」，最贴合 PRD §1.5「温柔克制」原则。
2. **实心材质 + 灰阶层次兜底质感**：靠三级灰阶、分隔线与极轻柔阴影区分控件层与内容层，即使无品牌色、无玻璃，层级与对比也清晰不平淡。
3. **降低决策成本、零返工升级**：令牌化后，未来引入品牌色只需把 `accent/primary` 语义令牌从「标签色」重映射成品牌橙，全局即可切换（见 §16）。

### 2.3 防止单调的 5 个手段

1. **主操作用「标签色填充」而非彩色**：➕ 按钮、主 CTA 在 Light 用近黑填充白字，Night 用近白填充黑字。
2. **靠灰阶层次 + 字重 + 留白撑节奏**：三级灰 + 大字重等宽数字金额。
3. **分类图标保留低饱和彩色圆底**：功能性识别色（非品牌色），让流水列表有生气。
4. **情感化插画 / 庆祝用暖色手绘**：空状态、首次记账、目标达成插画为暖色，作为「温度」的集中释放点。
5. **（可选）暖中性画布**：页面底可切换为微暖灰，整屏更柔（见 §4.3）。

> 一句话：**界面骨架黑白克制，颜色只给「钱」「分类识别」「庆祝时刻」。**

## 3. 令牌命名规范

- 统一格式：`类别/角色[/变体]`，全小写，斜杠分层。例：`semantic/expense`、`button.primary.bg`（组件令牌用点号表示「组件.部位.属性」）。
- 颜色原始令牌用「色相 + 明度档」：`gray/0` … `gray/1000`（数字越大越深）。
- 刻度型令牌用「类别/步进」：`space/4`（=16pt）、`radius/lg`。
- 代码导出建议：原始层与语义层导出为 TS 常量对象 `tokens` 与 `theme.light / theme.dark`（见 §18）。

## 5. 识别色与头像

### 5.1 分类图标识别色

- 每个记账分类配「图标 + 低饱和彩色圆底」。底色取**低饱和、明度统一**的一组色板，保证一排分类色彩和谐：

| 示例分类 | 令牌           | 圆底色（Light）      | 圆底色（Night，降明度保低饱和） |
| -------- | -------------- | -------------------- | ------------------------------- |
| 餐饮     | `cat/food`     | `#F4B183`（暖橘）    | `#C77E4F`                       |
| 交通     | `cat/transit`  | `#8FB7E0`（雾蓝）    | `#5E84AD`                       |
| 购物     | `cat/shopping` | `#E6A0B8`（藕粉）    | `#B36E86`                       |
| 居家     | `cat/home`     | `#A8C8A0`（草绿）    | `#74976E`                       |
| 医疗     | `cat/medical`  | `#9AB0E0`（淡靛）    | `#6A80AD`                       |
| 储蓄     | `cat/saving`   | `#C7B299`（暖驼）    | `#94806A`                       |
| 其他     | `cat/other`    | `gray/300`（中性灰） | `gray/300`                      |

> 此色板为「功能识别色」，**非品牌色**。图标本身为单色描边、可着色，圆底提供识别。

### 5.2 头像规则

- 用户未设置自定义头像时，统一使用**系统提供的默认头像占位**。
- **不使用取用户名首字生成的字符头像。**
- 头像为圆形（`radius/full`），用于顶栏入口、记账人标记、成员列表、成员贡献图。
- 多人家庭显示记账人头像；**单人家庭隐藏记账人字段**（PRD §4.4），不显示记账人头像。

## 6. 间距、网格与布局（L1 刻度）

- **基础栅格：8pt 网格**（4pt 为半步）。间距令牌：

| 令牌       | 数值 | 典型用途                          |
| ---------- | ---- | --------------------------------- |
| `space/0`  | 0    | 紧贴                              |
| `space/1`  | 4    | 图标-文字间隙、密集排版           |
| `space/2`  | 8    | 元素内间距                        |
| `space/3`  | 12   | 行内分组                          |
| `space/4`  | 16   | **页面左右安全边距 / 卡片内边距** |
| `space/5`  | 20   | 卡片间距                          |
| `space/6`  | 24   | 分区间距                          |
| `space/8`  | 32   | 大分区                            |
| `space/10` | 40   | 空状态留白                        |
| `space/12` | 48   | 顶部 hero 留白                    |

- **列表行高 ≥ 44pt**；**最小可点击区域 44×44pt**（`size/touchTarget`）。
- **同心圆角（Concentricity）**：iOS 26 强调嵌套圆角同心——外层卡片圆角大，内部元素圆角随内边距递减（内圆角 = 外圆角 − 内边距），视觉对齐。

## 8. 圆角与材质（实心，L1 刻度）

| 令牌          | 数值 | 用途                            |
| ------------- | ---- | ------------------------------- |
| `radius/sm`   | 8    | 标签、小按钮                    |
| `radius/md`   | 12   | 输入框、小卡片                  |
| `radius/lg`   | 16   | 内容卡片（默认）                |
| `radius/lg+`  | 20   | 大内容卡片 / 目标卡             |
| `radius/xl`   | 28   | 模态面板（记账 Sheet）顶部      |
| `radius/full` | 9999 | ➕ 按钮、头像、进度环、封面占位 |

**材质分层（全部实心，无玻璃 / 半透明 / 模糊）**：

- **控件层**：顶栏、Tab Bar、悬浮 ➕、Sheet 抓手区 → 实心背景（Light 用 `bg/base`，Night 用对应深色系统底），靠 `separator/opaque` 边线 + `elevation/1` 同内容层区分。
- **内容层**：卡片、流水行、报表 → 实心不透明，保证金额数字清晰可读。
- **海拔**：仅用分隔线 + 极轻柔阴影表达（见 §10），不用模糊、不用重投影。

### 8.1 描边令牌

| 令牌              | 数值                       | 用途               |
| ----------------- | -------------------------- | ------------------ |
| `border/hairline` | `StyleSheet.hairlineWidth` | 分隔线、列表行底边 |
| `border/thin`     | 1                          | 卡片边、输入框边   |
| `border/thick`    | 2                          | 选中态描边、进度条 |

## 9. 图标系统

- **系统 / 功能图标**：一律使用 **SF Symbols**（粗细、对齐、深浅色自动适配）。
- **分类图标**：统一描边风格、圆角、单色可着色的图标族 + §5.1 低饱和彩色圆底。
- **品牌图标**：App 图标、空状态 / 庆祝插画采用**暖色调手绘风格**（家、屋檐、存钱罐、一家人），传递温度。

| `icon/sm`   | 16   | 行内角标、辅助 |
| `icon/md`   | 20   | 列表次要图标   |
| `icon/base` | 24   | Tab、通用图标  |

## 10. 海拔与阴影（L1 刻度，克制）

| 令牌          | 阴影规格               | 用途                                         |
| ------------- | ---------------------- | -------------------------------------------- |
| `elevation/0` | 无阴影，仅 `separator` | 内容层卡片默认（贴底）                       |
| `elevation/1` | `y=1, blur=8, 黑 8%`   | 控件层（顶栏底边、Tab Bar 顶边）、列表浮起卡 |
| `elevation/2` | `y=2, blur=16, 黑 10%` | 悬浮 ➕ 按钮、Sheet 整体浮起                 |

> **Night 模式不靠阴影分层**（深底上阴影几乎不可见）：改用**更亮一档的表面色**表达海拔（`bg/card #1C1C1E` < `bg/elevated #2C2C2E`）+ 分隔线。这是 iOS 深色海拔的标准做法，工程实现需在 Night 下把阴影 opacity 降到 ≤ 4%。

## 11. 不透明度与层级（L1 刻度）

### 11.1 不透明度

| 令牌               | 值        | 用途                              |
| ------------------ | --------- | --------------------------------- |
| `opacity/disabled` | 0.35      | 禁用态控件                        |
| `opacity/pressed`  | 0.12      | 按压叠加层（覆盖在控件上的黑/白） |
| `opacity/scrim`    | 0.40      | Sheet / 弹窗背后的遮罩（纯黑）    |
| `opacity/skeleton` | 0.08–0.16 | 骨架屏闪烁区间                    |

> 说明：禁止用半透明做「玻璃」材质（§1）；不透明度仅用于**状态叠加、遮罩、骨架**这类非材质用途。

### 11.2 层级（z-index，用于浮层管理）

`z/content 0` < `z/sticky 10`（吸顶分组头）< `z/banner 20` < `z/fab 30`（悬浮 ➕）< `z/scrim 40` < `z/sheet 50` < `z/toast 60` < `z/alert 70`。

## 12. 动效与触感（L1 令牌）

- **遵循 iOS 26 标准转场**（Sheet 上滑、Push 等系统转场）。
- **动效令牌**：

| 令牌                 | 值                           | 用途                   |
| -------------------- | ---------------------------- | ---------------------- |
| `motion/tap`         | 100ms                        | 按压反馈               |
| `motion/micro`       | 150ms                        | 小元素出现 / 选中      |
| `motion/base`        | 250ms                        | 标准过渡、Sheet 内切换 |
| `motion/emphasized`  | 350ms                        | 页面 / Sheet 转场      |
| `motion/celebration` | ≤ 1500ms                     | 庆祝动效上限（§14.4）  |
| `easing/standard`    | `cubic-bezier(0.2, 0, 0, 1)` | 通用                   |
| `easing/decelerate`  | `cubic-bezier(0, 0, 0, 1)`   | 入场                   |
| `easing/accelerate`  | `cubic-bezier(0.3, 0, 1, 1)` | 出场                   |

- **Haptics**：保存成功（`success`）、达成目标（`success` 重）、删除（`warning`）、滑动确认到位（`impact`）。
- **减弱动态（Reduce Motion）**：开启时，所有 `motion/*` 退化为即时切换或交叉淡入；庆祝粒子改为静态文案，时长归零。

## 13. 可访问性（强约束）

- 文本对比度 ≥ WCAG AA（正文 4.5:1，大字 3:1）。
- **收支不只靠颜色**：必带 `+/-` 与文案。
- 分类色为**辅助识别**，不可作为唯一区分手段（必带名称 / 文字）。
- 全面支持 **Dynamic Type、VoiceOver、减弱动态、Light / Night 两种模式**。
- 实心材质下，Light 与 Night 两套配色均需各自验证文本对比度达标（含 Warm 画布变体）。
- 触控目标 ≥ 44×44pt（`size/touchTarget`）。

## 14. 核心组件规范（L3 组件令牌）

> 每个组件列出消费的语义令牌与状态变体。组件令牌**只引用 §4–§12 的 L2/L1 令牌**，不写裸 hex。

### 14.1 导航

- **顶栏**：实心材质（`separator/opaque` 底边 + `elevation/1`），左标题（Large Title 滚动收起），右上角圆形头像入口（进入「我的 / 设置」，与 IA §2 一致）。
- **底部 Tab Bar**：实心材质（顶边 `separator/opaque`），3 Tab（首页 / 报表 / 家庭）居左，**独立圆形 ➕ 悬浮按钮置右**（`accent/primary` 填充 + `elevation/2`）。这是品牌识别点。
- **远期 4 Tab**：见 §16 扩展预案。

### 14.2 按钮（状态表）

| 部位 / 令牌  | Primary                 | Secondary      | Destructive                 |
| ------------ | ----------------------- | -------------- | --------------------------- |
| `bg`         | `accent/primary`        | `accent/tint`  | 透明（文字 `state/danger`） |
| `text`       | `accent/onPrimary`      | `text/primary` | `state/danger`              |
| `radius`     | `radius/md`             | `radius/md`    | `radius/md`                 |
| `height`     | 50（min 44）            | 50             | 50                          |
| **pressed**  | 叠加 `opacity/pressed`  | 同左           | 同左                        |
| **disabled** | 整体 `opacity/disabled` | 同左           | 同左                        |

### 14.3 记账面板（流程 2，最高频）

- 从底部弹出的 **Sheet（实心，顶部抓手 + `radius/xl`）**，背后 `opacity/scrim`，支持半高 / 全高。
- 顶部「支出 / 收入」分段控件（Segmented，选中段 `accent/tint` + `text/primary`）。
- **大号金额输入区**（`type/amountHero`）+ 自定义数字键盘（金额是唯一必填，视觉绝对主角）。
- 分类横向滚动选择（`cat/*` 圆底）；备注 / 时间 / 记账人折叠为次要行。
- **单人家庭隐藏「记账人」字段**；多人家庭显示记账人（带默认头像，PRD §4.4）。
- 保存按钮：金额 > 0 才高亮可用（Primary 态），否则 disabled 态。

### 14.4 流水列表行（首页）

- 左：分类图标（带 §5.1 圆底，`icon/lg` on `radius/full`）；中：分类名（`type/body`）+ 备注（`type/subheadline` `text/secondary`）；右：金额（`type/amountRow`，收入 `semantic/income` `+`、支出 `semantic/expense` `-`）。
- 多人家庭：备注行带记账人默认头像小标记。
- 按日期分组，分组头（`type/footnote`，可吸顶 `z/sticky`）含当日小计。
- **左滑出编辑 / 删除**（权限不足不出现，PRD §12；删除走 §14.8 滑动确认）。

### 14.5 金额展示规范（v0.2 新增）

记账类 App 的「主角」，统一一套：

- **符号**：支出 `-`、收入 `+`，颜色取语义色；结余/中性取 `text/primary`。
- **整数 / 小数分级**：整数部分用主字号主字重；**小数部分（含小数点）降一档字号 + 字重**（如 hero 金额 40→28、Bold→Regular），高级且不抢读。
- **千分位**：每三位加分隔。**等宽数字（Tabular）必开**。
- **币种位预留**：货币符号在金额左侧，预留独立槽位（多币种远期，见 §16）。

### 14.6 进度类组件（预算 / 储蓄目标，P1）

- 预算：线性进度条（`border/thick` 粗细），≤80% 用 `text/secondary`，80% 转 `state/warning`，100%+ 转 `state/danger`。
- 储蓄目标：卡片（`radius/lg+`）含封面图 + 圆形 / 线性进度 + 剩余天数；达成时进度环 `state/success` 高亮 + 触发 §14.4-celebration。

### 14.7 图表（报表）

- 分类占比：环形图（Donut），中心显示总额；图例 = `cat/*` 色 + 名称 + 金额 + 占比。
- 成员贡献：横向条形图，统一用 `accent/tint` 填充条 + `text/primary` 数值，靠成员名称与数值区分（PRD §11.5）。
- 趋势折线 / 收支双柱（P1）：支出 `semantic/expense`、收入 `semantic/income`。
- **储蓄类流水不进消费占比 / 趋势图**（PRD 口径）。

### 14.8 反馈与提示

- **顶部条幅 Banner**（P1：预警 / 月度总结）：实心条 + 左语义图标 + 文案 + 关闭。预警 `state/warning`、超支 `state/danger`、总结中性暖。
- **二次确认（破坏性）**：弹窗 + 输入校验（手机号后 4 位 / 家庭名）+ **滑动确认控件**（PRD §7/§8「滑动以确认」单独成组件，到位触发 `impact` haptic）。
- **Toast / 同步态**：离线「已保存，稍后同步」用 `state/info` 轻提示（`z/toast`）。
- **空状态**：暖色手绘插画 + 一句温暖引导 + 主操作按钮（如「记下第一笔」），留白用 `space/10`。

### 14.9 暖心时刻 Celebration（v0.2 新增，统一规范）

把分散的庆祝点统一为同一套「家账情绪签名」，三处共用：

| 触发点（PRD）       | 视觉                             | 文案来源                    |
| ------------------- | -------------------------------- | --------------------------- |
| 首次记账（§4.3）    | 面板内轻量缩放 + 礼花            | 固定欢迎文案                |
| 储蓄目标达成（§9）  | 全屏进度环高亮 + 礼花 + 暖色插画 | 目标相关文案                |
| 月度总结生成（§11） | 首页条幅 + 进入总结卡            | 暖心文案池随机（PRD §11.8） |

- 统一规格：暖色手绘插画 + 克制粒子，时长 ≤ `motion/celebration`（1.5s），`success` haptic。
- **尊重减弱动态**：粒子退化为静态插画 + 文案，不影响信息传达。
- 度的把握：庆祝是「点到为止的仪式感」，不阻断主流程、不强制全屏停留。

## 15. UI 模式（Light / Night）

- v0.2 即统一提供 **Light / Night 两种模式**，跟随系统或手动切换，所有令牌提供两套映射（见 §4、§18）。
- 主强调色自动反相（Light 近黑 / Night 近白）。
- Night 模式下：背景用 iOS 系统深色语义底；分类色降明度、保低饱和；**海拔改以更亮表面色表达**（§10）；阴影 opacity ≤ 4%。
- Warm 画布变体（§4.3）仅作用于 Light（Night 两者一致）。
