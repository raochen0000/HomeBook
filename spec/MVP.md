## 1. MVP 目标

第一版（v0.1）的目标是**跑通「家庭协作记账」的核心闭环**，验证产品定位是否成立，而非堆功能。

### 1.1 核心假设（待验证）

- 用户愿意以「家庭」为单位共同记账，而非各记各的。
- 「一家人共享一个账本」的协作体验，比个人记账工具更有粘性。

### 1.2 成功标准

MVP 阶段**暂不设量化指标基线**。验收标准为：**核心链路（登录 → 记账 → 家庭协作 → 基础报表）可正常使用、无阻断性缺陷，能稳定上线即可。**

量化指标（如首次记账率、协作激活率、留存等）待上线后结合真实数据再行设定。

## 3. 明确「不做什么」（防范围蔓延）

MVP 阶段**明确不做**以下内容，即使开发中有余力也应克制：

- ❌ 自定义分类的增删改（先用系统预设分类）
- ❌ 周 / 年报表维度与多种图表
- ❌ 月度总结卡与暖心文案池
- ❌ 系统推送（仅做 App 内兜底通知）
- ❌ 账号注销、户主继任、多家庭
- ❌ 手机号 / Apple 以外的第三方登录（微信等后置）
- ❌ 任何导入 / 导出能力

## 4. 里程碑排期（按依赖关系分批）

> 排期为相对批次，非绝对工期；每批结束应可独立验收。

### M0 · 工程地基（P0 第零批）

- 脚手架（Expo + TypeScript）、设计令牌主题（Light / Night）、本地 DB、API 层骨架
- 底部导航（3 Tab + 独立 ➕）、`<Money/>` 金额组件、空状态等通用组件
- 依赖：TECH.md 技术选型与工程结构、DESIGN.md 设计令牌

**验收**：可运行的空壳应用，主题与导航骨架就位，本地 DB 与 API 层（含 Mock）可联调。

### M1 · 账号与记账地基（P0 第一批）

- 流程 1 登录 + 单人家庭自动创建
- 流程 2 快速记一笔
- 流程 10 编辑 / 删除流水
- 依赖：DATAMODEL 中 USER / FAMILY / MEMBERSHIP / TRANSACTION / CATEGORY（预设分类）

**验收**：单人可完整记账、改账、删账，数据正确入库。

### M2 · 家庭协作闭环（P0 第二批）

- 流程 3 邀请 + 流程 4 扫码加入
- 流程 5 转让 / 退出 / 解散
- 流程 13 关键通知（被移除 / 转让的 App 内提示）
- 依赖：INVITATION、MEMBERSHIP 状态流转、family_id 绑定与离线一致性

**验收**：多人可加入同一家庭共享账本，家庭生命周期可正常流转，无串账。

### M3 · 基础报表（P0 第三批）

- 流程 9 基础版报表（本月收支结余 + 分类占比）

**验收**：用户能看到「这个月家里花了多少 / 各分类占比」。

### M4 · 增值功能（P1）

- 流程 11 分类管理 → 流程 8 预算 → 流程 7 储蓄目标 → 流程 9 完整报表 + 月度总结 → 流程 6 移除成员 → 流程 13 通知体系完整版

**验收**：差异化与精细化能力上线，按数据反馈调整优先级。
